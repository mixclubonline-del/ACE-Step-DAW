# ACE-Step DAW - Story Matrix

> Canonical QA index for product verification.
> Source of truth order: capability doc -> story matrix -> automated/manual execution artifacts.

## Coverage Labels

- `A-full`: fully automated with stable coverage
- `A-partial`: partially automated, still needs manual or follow-up checks
- `H-required`: human ear/eye/judgment required
- `Blocked-env`: cannot run without API, device, or external environment
- `Missing`: no current coverage

## Agent Operating Rules

For each story, QA agents should start from this matrix and only then open linked docs/tests.

- Read the capability doc first for setup, boundaries, and expected evidence.
- Run the linked automated tests in the listed order.
- Respect the `Agent Setup` rule before using `window.__store` or other browser globals.
- Treat `Browser-only` as a user-visible flow that must be verified through DOM interaction.
- Treat `H-required` as mandatory human follow-up, not optional debt.
- Attach screenshots, Playwright traces, console output, or short GIFs according to the capability doc.
